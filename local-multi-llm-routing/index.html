Active Project Started May 2026

Local Multi-LLM Routing

Turning the main agent into an intelligent controller that dynamically routes tasks to the best available local LLMs.

Project Goal

Enable the primary AI agent to act as a smart controller that evaluates tasks in real time and routes them to the most suitable local LLM based on the nature of the work — whether that means deep research, long-context generation, cost efficiency, or privacy-sensitive operations.

Key Capabilities

Real-time Routing

The controller evaluates each request and decides whether to use the main model or route to a local LLM.

Cost Optimization

Heavy token usage is offloaded to local models, significantly reducing API costs over time.

Privacy-First Workflows

Sensitive or personal tasks can be kept entirely on local infrastructure.

Specialized Agent Teams

Foundation for building teams of agents, each optimized for different strengths and models.

Implementation Approach

Phase 1 — Router Skill

Create a dedicated `local-llm-router` skill that contains clear decision criteria for when to use local models versus the primary model.

Phase 2 — Secure Connection

Establish a reliable and secure connection between the main agent and local LLM instances (via tunnels or co-location).

Phase 3 — Testing & Iteration

Run real-world tests across research, writing, and analysis tasks to refine routing accuracy.

Current Status

Planning Phase

Detailed implementation plan and assessment report completed. Awaiting decision to begin building the first version of the router skill.
